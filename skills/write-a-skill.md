# write-a-skill

Write one new skill file in `skills/`, short enough to read in a breath and shaped
like the others.

1. **Name one job.** Write the `name` (kebab-case, matching the filename) and a
   one-line `description` that says when to reach for it. One job only; if it needs
   "and", split it into two skills.
2. **Frame it.** Open with one sentence stating the job, reusing the shared names
   from GLOSSARY.md so it fits the domain (createGame, nextState, collides, issues/).
3. **List the steps.** Write 3 to 5 numbered steps, each a **bold lead** word and a
   plain instruction. Point at the real tools (`node --test`, `issues/`, AGENTS.md),
   not generic advice.
4. **Close with a principle.** End on one line that states the rule the skill exists
   to enforce, not another step.

If it does not fit on one screen, it is doing two jobs; cut it down.
